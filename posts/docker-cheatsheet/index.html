Docker - Cheatsheet

Rob
 2022-07-26  293 words   2 minutes 
Contents

Docker is a popular platform for building, deploying, and running applications using containers. It allows developers to package their applications and dependencies into lightweight containers, which can then be easily deployed and run on any platform that supports Docker.

There are several docker commands that are commonly used to manage and interact with Docker containers and images. Here are some of the most useful and frequently used docker commands:

docker run: This command is used to run a new container from an image. For example:

docker run -d --name my-web-server -p 80:80 nginx

This command will run a new container based on the nginx image, and it will expose port 80 on the host machine to port 80 on the container. The -d flag runs the container in detached mode, and the –name flag assigns a name to the container.

docker ps: This command lists all the running containers on your system. You can use the -a flag to list all containers, including those that have stopped. For example:

docker ps -a

docker stop: This command stops a running container. For example:

docker stop my-web-server

docker start: This command starts a stopped container. For example:

docker start my-web-server

docker rm: This command removes a container from your system. For example:

docker rm my-web-server

docker pull: This command downloads an image from a Docker registry. For example:

docker pull ubuntu

docker images: This command lists all the images on your system. For example:

docker images

docker rmi: This command removes an image from your system. For example:

docker rmi ubuntu

These are just a few examples of the many docker commands that are available. By learning and using these commands, you can easily manage and interact with Docker containers and images on your system.
