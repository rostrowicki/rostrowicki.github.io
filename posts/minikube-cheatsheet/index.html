Minikube Cheatsheet

Rob
 2022-07-26  315 words   2 minutes 
Contents

Minikube is a tool that allows you to run a single-node Kubernetes cluster locally on your machine. It is a great way to learn and experiment with Kubernetes without the need for a large, complex cluster.

One of the key concepts of Minikube is the virtual machine (VM) that it runs on. Minikube uses a VM to create a self-contained environment in which to run the Kubernetes cluster. By default, Minikube uses the VirtualBox hypervisor to create the VM, but it can also use other hypervisors such as VMware Fusion and Hyper-V.

Another important concept in Minikube is the Kubernetes dashboard. The dashboard is a web-based interface that allows you to view and manage the resources in your cluster. You can use the dashboard to deploy and manage applications, view the status of your resources, and access the logs of your containers.

To start using Minikube, you first need to install it on your machine. Once it is installed, you can use the “minikube start” command to create and start the Minikube VM. For example:

minikube start

This will create and start the Minikube VM, and it will also start the Kubernetes cluster inside the VM.

You can then use the “minikube dashboard” command to access the Kubernetes dashboard.

minikube dashboard

This will open the dashboard in your web browser, allowing you to view and manage the resources in your cluster.

To stop Minikube, you can use the “minikube stop” command.

minikube stop

This will shut down the Minikube VM and the Kubernetes cluster.

In summary, Minikube is a tool that allows you to run a single-node Kubernetes cluster locally on your machine. It uses a virtual machine to create a self-contained environment, and it also includes a dashboard for managing and interacting with the cluster. By using Minikube, you can easily learn and experiment with Kubernetes without the need for a large, complex cluster.
